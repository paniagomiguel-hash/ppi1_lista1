Protocolo HTTP

O protocolo utilizado para transferir páginas da web entre um servidor (local de hospedagem do site) e navegador (usuário que acessa o site) é o HTTP (Hypertext Transfer Protocol).

Ele integra a camada de aplicação (camada 7) do modelo OSI (Open Systems Interconnection) e é sem estado, o que significa que cada requisição é independente. O servidor não retém informações sobre interações anteriores, a menos que sejam utilizados cookies ou sessões.

Como o HTTP opera?

  1. O cliente (navegador) faz uma solicitação ao servidor web.

  2. O servidor retorna com:
    ● Um código de status (por exemplo, 200 = sucesso, 404 = não encontrado);
    ● E o conteúdo requisitado (HTML, imagens, vídeos etc.).

  3. O protocolo TCP/IP é utilizado para a comunicação, normalmente por meio da porta 80.

  4. Cada mensagem contém:
    ● Cabeçalho (Header) → inclui dados técnicos, como tipo de conteúdo, idioma, cookies etc.;
    ● Corpo (Body) → armazena os dados reais (como, por exemplo, o código HTML da página).


Principais tipos de requisição HTTP:

Método Função Exemplo de uso
GET Solicita dados do servidor Carregar uma página
HEAD Solicita apenas o cabeçalho da resposta Verificar informações sem baixar o conteúdo
POST Envia dados para o servidor Enviar um formulário
PUT atualiza ou cria um recurso Editar informações
PATCH Modifica parcialmente um recurso Atualizar um campo específico
DELETE Remove um recurso do servidor Excluir uma postagem
CONNECT Cria um túnel de comunicação Usado no HTTPS
OPTIONS Mostra quais métodos estão disponíveis Testes e diagnósticos
TRACE Mostra o caminho da requisição ate o servidor Usado para depuração


HTTP x HTTPS

O HTTP não possui criptografia, ou seja, os dados trafegam em texto puro — o que é inseguro. Por isso existe o HTTPS (HTTP Secure), que usa SSL/TLS para criptografar e autenticar a conexão, garantindo segurança.

● O HTTPS usa a porta 443 e aparece com um cadeado no navegador.

● Sites com HTTPS são considerados mais seguros e têm melhor ranqueamento no Google (SEO).


Resumo Final

O HTTP é o protocolo que permite a comunicação entre navegadores e servidores na web, enviando e recebendo informações.
Ele é essencial para o funcionamento da Internet, mas por ser inseguro, foi aprimorado pelo HTTPS, que garante sigilo e autenticidade dos dados.
Além de carregar sites, o HTTP também é usado em aplicativos, APIs e sistemas de arquivos online.

Requisições

Quando um usuário acessa um site, o navegador envia uma requisição HTTP (request) ao servidor, pedindo informações específicas, como uma página ou dados de um formulário. O servidor, então, processa essa solicitação e responde com uma resposta HTTP (response), que contém o conteúdo solicitado e informações sobre o status da requisição.
Os requests e responses possuem diversos campos (headers e body) que controlam o tipo de dado, formato, idioma, códigos de status e muito mais. A compreensão desses campos é fundamental para entender como a comunicação entre cliente e servidor acontece na web.

Exemplo de requisição HTTP (Request)


    GET /pagina.html HTTP/1.1
    Host: www.exemplo.com
    User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
    Accept: text/html,application/xhtml+xml
    Accept-Language: pt-BR,pt;q=0.9
    Connection: keep-alive

Campo Função
GET /pagina.html HTTP/1.1 Linha inicial (request line). Indica método (GET), recurso (/pagina.html) e versão do HTTP (1.1)
Host Especifica o domínio do servidor que você quer acessar. Obrigatório no HTTP/1.1
User-Agent Identifica o navegador e sistema operacional que está fazendo a requisição
Accept Indica os tipos de conteúdo que o cliente aceita (HTML, JSON, etc.)
Accept-Language Linguagem preferida do conteúdo
Connection Controla se a conexão TCP será mantida aberta ou fechada após a resposta

Exemplo de resposta HTTP (Response)


    HTTP/1.1 200 OK
    Date: Sun, 20 Oct 2025 14:00:00 GMT
    Server: Apache/2.4.41 (Ubuntu)
    Last-Modified: Mon, 14 Oct 2025 10:00:00 GMT
    Content-Type: text/html; charset=UTF-8
    Content-Length: 1024
    Connection: keep-alive

    <html>
    <head>
    <title>Exemplo de Página</title>
    </head>
    <body>
    <h1>Bem-vindo!</h1>
    <p>Esta é uma página de exemplo.</p>
    </body>
    </html>
Campo Função
HTTP/1.1 200 OK Linha inicial (status line). Código de status (200 = sucesso) e mensagem textual
Date Data e hora em que o servidor respondeu
Server Informação sobre o software do servidor
Last-Modified Data da última modificação do recurso
Content-Type Tipo do conteúdo retornado (HTML, JSON, etc.)
Content-Length Tamanho em bytes do corpo da resposta
Connection Mantém ou encerra a conexão TCP

Dicas sobre request e response

  • Você pode usar o inspector do navegador:
    No Chrome/Edge: F12 → Network → Atualizar página → clicar em uma requisição.
    Mostra request headers, response headers e body.

  • Ferramentas como Postman ou curl também permitem testar e visualizar requisições detalhadas.
